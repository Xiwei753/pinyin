# IME Contract (核心契约)

本契约定义了输入法前端（Linux/Android）与核心引擎之间的责任划分和数据交换标准。

## 责任划分

- **前端（Frontend）**只负责：
  - 界面展示（键盘、候选栏、设置页面）
  - 动画效果
  - 按键事件捕获和转发
  - 候选词的展示和用户选择结果的反馈

- **核心（Core）**负责：
  - 拼音解析（全拼、双拼等）
  - 九键（T9）解析
  - 候选词生成与排序
  - 词频和用户习惯的记忆
  - 词库管理
  - 自定义短语处理

## 跨端架构说明

- Linux 和 Android 前端可以完全分开写，针对不同平台做特定的优化。
- 但两端**必须**遵守同一套核心契约，保证输入体验的一致性。
- 第一阶段核心实现采用 **Rime (中州韵)** 引擎来快速落地。
- 未来如果 Rime 在某些定制化场景下（如极复杂的自定义九键逻辑或特殊的联想策略）不够用，可以把 `core/` 下的逻辑替换成自研核心，但需要保证前端契约尽量不变。

## 契约定义细节

详见同目录下的格式文档：
- 前端输入什么 / 按键事件格式：`key_event_format.md`
- 核心返回什么 / 候选词格式：`candidate_format.md`
- 功能开关如何描述：`feature_flags.md`
- 全拼模式和九键模式的区别和切换逻辑，将由特定的 `feature_flags` 和输入方案决定。
