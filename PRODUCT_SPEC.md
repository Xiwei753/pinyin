# 产品规格说明书 (PRODUCT_SPEC.md)

本文档定义了本项目的核心产品目标、功能规范、UI/UX 要求和开发阶段路线图。所有的功能开发和 Bug 修复必须以此文档为指导原则。

## 1. 项目目标与路线图

### 最终目标
构建一个跨 Linux 和 Android 平台的中文输入法。

### 第一阶段目标与优先级
**重点聚焦：Android 九键中文输入法**。当前阶段仅关注如何让九键在日常使用中足够稳定且好用。Linux 版本同步、Android 全键等其他内容不在第一阶段的考虑范围内。

**第一阶段优先级：**
1. **候选正确**：不乱喷候选词，常用词和常用字完整且准确。
2. **打字速度**：响应必须迅速，绝对不能有打开键盘时的卡顿。
3. **词库丰富**：覆盖日常交流所需的高频词汇。
4. **稳定不崩**：核心逻辑不出现闪退。
5. **UI 能看即可**：现代圆角、深浅色可用即可，不追求复杂炫酷的动画。

**最不能接受的问题：**
- 候选乱喷（如输入 `不太行` 出现 `不太新股`）。
- 常用单字缺失。
- 打开键盘卡顿。
- 显示裸数字而不显示拼音。
- 0 键不上屏。
- 设置修改后无效。

## 2. 核心输入与候选规则

### 核心架构
**九键数字 -> 拼音音节解码 -> 拼音 preedit -> 拼音查字词 -> 中文候选**
- **严禁**使用“数字 prefix 直接查中文词库”的回退方案。

### 候选规则
- **宁可少，不要乱喷**：在半截拼音状态下，只显示极少量安全候选，不允许进行长词联想。
- **单音节处理**：完整拼音的单音节输入必须以**单字**为主，不要混入多字词（如 `wo` 候选以“我”为主，不要出现“我们”这种组合）。
- **多音节处理**：完整拼音时优先展示完整词汇；如果没有完整词汇，则降级为短词或单字组合。
- **只做短语组合**：第一阶段不做真正的整句语言模型。
- **裸数字 Fallback**：数字回退选项永远放在候选列表末尾，绝不允许抢占中文词汇的位置。
- **候选数量**：默认显示 5 个候选，并允许用户在设置中切换为 5 / 10 / 15 / 30。
- **候选栏交互**：第一阶段不做候选翻页，使用横向滑动浏览候选。

### 回归必测用例
- `96` -> preedit `wo`，候选含“我”且靠前。
- `64` -> preedit `ni`，候选含“你”。
- `82` -> preedit `ta`，候选含“他/她”。
- `33` -> preedit `de`，候选含“的/得/地”。
- `744` -> preedit `shi`，候选含“是”。
- `28` -> preedit `bu`，候选含“不”。
- `28824` -> preedit `bu tai`，候选含“不太”。
- `288249464` -> preedit `bu tai xing`，第一候选必须是“不太行”。(不应该出现“不太新股”或“不太英语”)
- `546842692674264` -> preedit `jin tian wan shang`，候选包含“今天晚上”。
- `28 1 824 1 9464` -> preedit `bu tai xing`，候选包含“不太行”。

## 3. 按键与显示规则

### 特殊按键
- **`1` 键**：拼音音节分隔符/分词键。
  - 不参与字母映射。
  - 有 buffer 时，插入音节分隔符。
  - 空 buffer 时，无操作（暂时不要绑定为符号面板）。
- **`0` 键**：视觉上表现为中文状态下的空格键/空白键，不要强调数字“0”。
  - buffer 为空时，输入空格。
  - buffer 有候选时，提交第一候选词。
- **删除键**：
  - 有 buffer 时，删除 buffer 中的内容。
  - buffer 为空时，删除输入框内容。
  - 长按连续删除和上滑清空功能留待后续开发。

### Preedit（预编辑区）展示
- **核心原则**：主显示拼音 preedit，绝不主显示裸数字。
- **示例**：
  - `96` 显示 `wo`
  - `64` 显示 `ni`
  - `288249464` 显示 `bu tai xing`
  - `546842692674264` 显示 `jin tian wan shang`
- （点击拼音段落进行编辑的功能留待后续支持）

### 英文/数字/符号
- **英文九键**：列入后续规划，当前非重点（后续考虑滑动空格切换中英文）。
- **数字模式**：单独实现数字模式，与中文拼音严格分离。
- **符号面板**：提供中文标点和符号面板（例如通过 `*` 键或单独的符号键触发），**不要**绑定到空 buffer 的 `1` 键。
- **Emoji**：第一阶段不做。

## 4. 词库使用规范

- **来源**：使用 `rime-ice` 作为开源词库来源。必须在项目中保留 GPLv3 许可证及 `rime-ice` 的来源说明。
- **字典区分**：
  - `base.dict.yaml`：用作主要的高频常用词语词库。
  - `8105.dict.yaml`：**仅作为单字候选表使用**，必须保证单字能够被打出，且常用字优先，但禁止将其当作主词库疯狂推荐冷僻字。
- **内部格式**：保持使用 `词语<TAB>拼音<TAB>权重` 的形式。
- **九键映射**：九键码必须由拼音自动生成，禁止在字典中手写数字码。

## 5. UI、手感与设置

### UI 与交互
- 风格上采用现代圆角设计，适配深浅色模式，不追求复杂的交互动画。
- 第一阶段动画只保留按键按下的 `pressed` 状态和相关接口。
- 候选栏采用单行横向滑动。
- 视觉排版：拼音 preedit 区域字体适中，中文候选词更醒目。
- 支持键盘高度调整（低 / 标准 / 高）。

### 触感反馈
- 振动**必须使用**系统标准的 `View.performHapticFeedback()` 接口。
- 振动默认开启，但必须完全尊重系统的触感设置。
- 允许在 App 设置中关闭振动。
- **禁止**自定义写入 Vibrator 振动波形。
- **禁止**制作按键音。

### 设置 App 功能
- APK 中必须保留桌面入口（Launcher Activity）。
- **首页需求**：启用输入法、切换输入法、设置、词库状态、关于。
- **设置页首批需求**：触感反馈开关、候选数量选项、主题切换、键盘高度调整、词库状态展示、关于与许可证说明。
- 设置必须统一通过 `SettingsRepository` 进行管理。

## 6. 隐私与兼容性限制

### 隐私与网络
- **零联网**：输入法本体必须完全离线运行。
- **不上传**：绝不收集或上传用户的输入内容。
- **禁止功能**：第一阶段**绝不添加**云词库、在线 AI 预测、账号同步等功能。用户私人词库未来将考虑使用纯本地加私有 GitHub 仓库的形式同步。

### 兼容性
- 优先支持现代 Android 设备（用户的最旧测试设备为 Android 13）。
- 不为过时的老版本 Android 增加额外开发负担。
- **禁止**为 x86 Android 进行特殊适配。
- 横屏模式和平板适配不在第一阶段的考虑范围内。
