# 仓库目录结构说明

本文件简要介绍 `personal-ime-core` 仓库下的目录结构以及每个文件的用途。

## 目录与文件总览

```text
personal-ime-core/
├── LICENSE                    # 仓库的开源协议文件（如果存在）
├── NOTICE.md                  # 仓库的注意事项或补充声明文件（如果存在）
├── README.md                  # 项目的主说明文档，包含简介、功能说明、运行和测试指南
├── STRUCTURE.md               # 本文件，介绍仓库的目录结构和各文件用途
├── data/                      # 存放字典数据的目录
│   ├── base_dict.txt          # 基础词库文件，格式为: 拼音<TAB>词语<TAB>词频
│   └── user_freq.json         # 用户词频记录文件，用于存储用户选择候选词的次数，以调整排序
├── ime_core/                  # 核心引擎代码包
│   ├── __init__.py            # 将目录标记为 Python 包，并可用于导出核心类
│   ├── dictionary.py          # 负责解析和加载 `data/base_dict.txt` 的基础词库模块
│   ├── engine.py              # 输入法核心引擎模块，负责拼音到候选词的匹配和排序计算
│   └── user_freq.py           # 负责读取和保存用户词频记录 `data/user_freq.json` 的模块
├── main.py                    # 命令行交互的主程序入口，提供基于终端的原型测试界面
└── tests/                     # 单元测试目录
    └── test_engine.py         # 针对 `ime_core` 的单元测试，包含词库加载、引擎匹配和用户词频保存等测试用例
```

## 各模块详细说明

### `data/` 目录
提供输入法所需的数据支持。
- **`base_dict.txt`**：静态基础词库。
- **`user_freq.json`**：动态的用户词频记录，用于实现越用越顺手的功能。

### `ime_core/` 目录
输入法引擎的核心逻辑所在，各个组件职责单一。
- **`dictionary.py`**：解析文本词典，提供按拼音查询基础候选词的能力。
- **`user_freq.py`**：处理 JSON 格式的词频持久化。
- **`engine.py`**：组装词典和词频，提供 `candidates()` 方法获取排序后的候选词列表，并提供 `select()` 方法更新词频。

### 根目录文件
- **`main.py`**：这是一个简单的命令行前端，实例化引擎并实现一个不断接受输入、展示候选、选择候选的交互循环。
- **`tests/test_engine.py`**：通过 `pytest` 编写的自动化测试，确保核心逻辑的正确性和稳定性。
