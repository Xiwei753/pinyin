# 真机验收测试指南

## 准备工作

1. 安装 APK 到真机：`./gradlew installDebug`
2. 在系统设置中启用 XiweiT9 输入法，并切换为当前输入法
3. （可选）开启调试日志：在 XiweiT9 设置中打开「调试日志」开关，然后用 `adb logcat -s XiweiT9EditorPolicy XiweiT9StateMachine XiweiT9Debug` 查看日志

## 测试场景

### 1. 普通中文文本框

用系统备忘录/便签应用测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1.1 | 输入 `96` | preedit 显示 `wo`，候选栏出现「我」 |
| 1.2 | 点击候选「我」 | 「我」提交到文本框，preedit 消失 |
| 1.3 | 输入 `288249464` | preedit 显示 `bu tai xing`，第一候选是「不太行」 |
| 1.4 | 在输入框有候选时按 `0` | 第一候选被提交 |
| 1.5 | 清空输入框，按 `0` | 输入一个空格 |
| 1.6 | 输入 `96` 后有 buffer 时按退格 | buffer 被删除，preedit 消失 |
| 1.7 | buffer 为空时按退格 | 文本框中的文本被删除 |

### 2. 密码框

用带密码输入框的应用（或网页密码栏）测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 2.1 | 进入密码框 | 默认是英文模式，不显示中文候选，不显示拼音 preedit |
| 2.2 | 尝试切换到中文 | 无法切换到中文候选（中/英切换无效） |
| 2.3 | 输入字母/数字 | 只输出字符，无预编辑、无候选 |
| 2.4 | 按退格 | 正常删除已输入字符 |
| 2.5 | 输入内容不会进入用户词库 | 重启输入法后无记忆 |

### 3. 数字框 / 电话框

用带数字输入框的应用测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 3.1 | 进入数字框 | 默认显示数字键盘 |
| 3.2 | 按 `0` | 输入数字 `0` |
| 3.3 | 按 `2`–`9` | 输入对应数字 |
| 3.4 | 按退格 | 正常删除数字 |
| 3.5 | 切换模式 | 不显示中文候选 |

### 4. URL 栏

用浏览器地址栏测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 4.1 | 进入 URL 输入框 | 默认英文模式，不显示中文候选 |
| 4.2 | 输入 `www.example.com` | 能看到英文输入 |
| 4.3 | 输入 `.`、`/`、`@`、`-`、`_` 等符号 | 符号正常输入 |
| 4.4 | 打开符号页 | 符号页正常，不会回退到中文候选 |
| 4.5 | 尝试切换到中文 | 中文候选不出现 |

### 5. Email 框

用邮件应用或网页邮箱测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 5.1 | 进入 email 输入框 | 默认英文模式，不显示中文候选 |
| 5.2 | 输入 `user@example.com` | `@` 和 `.` 能正常输入 |
| 5.3 | 尝试切换到中文 | 中文候选不出现 |

### 6. 聊天发送框

用微信、QQ 等聊天应用测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 6.1 | 在聊天输入框按回车 | 发送消息（不仅换行） |
| 6.2 | 输入拼音后有 composing 时按回车 | 先提交文字，再执行发送 |
| 6.3 | 验证发送按钮 | 发送按钮图标为「发送」形状 |

### 7. 搜索框

用浏览器搜索栏或应用内搜索框测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 7.1 | 进入搜索框 | 键盘回车键显示「搜索」或放大镜图标 |
| 7.2 | 输入关键词后按回车 | 执行搜索操作 |

### 8. 多行文本

用备忘录或长文本输入框测试。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 8.1 | 输入文字后按回车 | 插入换行，不发送 |
| 8.2 | 连续输入多行 | 可正常换行 |

### 9. 符号页

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 9.1 | 在中文模式下按「符」键 | 进入符号页 |
| 9.2 | 点击一个符号（如 `。`） | 符号提交到文本框，符号页保持打开 |
| 9.3 | 符号页左下角 | 显示「中」或「英」 |
| 9.4 | 点击符号页左下角按钮 | 返回进入符号页之前的文本模式（中文或英文） |
| 9.5 | 在符号页按 `123` | 进入数字页 |
| 9.6 | 在符号页按空格 | 输入一个空格 |

### 10. 视觉布局验证

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 10.1 | 在普通中文文本框，观察左侧标点栏 | `， 。 ？ ！` 仍是竖向列表，按钮间距和九宫格间距一致 |
| 10.2 | 比较左侧标点按钮宽度和左下角“符”按钮宽度 | 两者宽度一致 |
| 10.3 | 进入符号页，观察左侧分类栏 | “标点 / 数学 / 括号 / 其他”仍是竖向列表 |
| 10.4 | 比较符号页分类按钮高度和右侧符号网格行高 | 每个分类按钮高度等于符号网格每一行的键高度 |
| 10.5 | 观察左下角按钮 | “符 / 中 / 英”与底部行 `123 / 空格` 高度一致、底部对齐 |
| 10.6 | 各个页面的 key 圆角一致 | 九宫格、左栏、右栏、底部行圆角大小统一 |

### 12. 调试日志验证

若开启调试日志，用 `adb logcat` 验证：

```
# 进入文本框时应有：
XiweiT9EditorPolicy: inputType=0x00000001 ... defaultKeyboardMode=ChineseT9 ...

# 切换模式时应有：
XiweiT9StateMachine: action=ToggleChineseEnglish beforeMode=ChineseT9 afterMode=EnglishT9 ...
```
