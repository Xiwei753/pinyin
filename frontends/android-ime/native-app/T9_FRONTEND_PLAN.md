# Android 原生九键前端设计文档

## 目标
- 开发一个独立的 Android 原生九键输入法前端。
- 使用 Kotlin 进行开发。
- 基于 Android 官方的 `InputMethodService` 框架。
- 第一阶段仅实现最小原型验证，不追求发布成完整的输入法产品。

## 最小原型功能

1. **界面展示**
   - 显示一个基础的 3x4 九键键盘 UI。
   - 按键布局如下：
     - `1` (无字母，可保留作符号或其他用途)
     - `2 ABC`
     - `3 DEF`
     - `4 GHI`
     - `5 JKL`
     - `6 MNO`
     - `7 PQRS`
     - `8 TUV`
     - `9 WXYZ`
     - `符号` (按键预留，可暂不实现具体功能)
     - `0`
     - `删除 / 回车`

2. **输入逻辑**
   - 点击 `2-9` 的数字键时，将对应的数字追加到当前的输入缓冲区，并在界面上显示（如预编辑区）。

3. **候选词处理**
   - 候选栏先不接入完整的 Rime 引擎，而是使用内置的最小本地字典作为 mock，验证前端与输入逻辑的连通性。
   - 最小测试码表匹配规则：
     - 输入 `64426` -> 候选显示“你好”
     - 输入 `748732` -> 候选显示“输入法”

4. **文本提交**
   - 点击候选栏中的候选词后，输入法需要将选中的中文文本通过 `InputConnection` 提交（commitText）到目标输入框中，并清空当前的输入缓冲区。

## 阶段说明
- 此阶段核心目的是“跑通前端手感和九键逻辑”。
- 不强制要求立刻对接复杂的 Rime 引擎 JNI。只要能弹出一个自己画的九键键盘，点击数字能出测试候选，点击候选能上屏，即视为第一阶段原型成功。
