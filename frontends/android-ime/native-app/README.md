# Android 原生应用 (规划中)

这里是未来 Android 原生输入法 App 的预留位置。

## 未来规划
目前的第一阶段，我们仅通过打包配置的方式，复用现成的 Android Rime 前端进行功能验证。
但如果在未来，现有的框架无法满足追求极致流畅的动画效果与打字手感，我们将考虑使用 Kotlin 或 Java 在此目录下实现独立的 Android 原生前端。

## 原生前端的职责
即使在未来开发完整的 Android App，该原生前端也只负责以下部分：
- 键盘界面与交互 UI
- 触摸反馈机制
- 极致流畅的动画表现
- 候选词栏渲染
- App 设置页面
- 数据同步的入口与鉴权

## 核心复用
无论原生 App 如何演进，其核心的输入能力（包括分词、词典查询、候选排序算法等）仍将优先复用 `shared` 层提供的跨端基层，保证多平台一致性。

**注意：** 当前阶段**不要**在此实现完整的 App。本目录及此文档仅用作未来演进的路线说明。
