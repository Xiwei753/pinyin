# 词库格式说明 (Dictionary Format)

本文档描述了 personal-ime-core 的第一版词库格式。

## 当前格式

第一版词库 `data/base_dict.txt` 使用简单的 TSV (Tab-Separated Values) 格式，每一行为一条词条，格式如下：

```
pinyin<TAB>word<TAB>freq
```

### 列含义

- **pinyin**: 拼音串。当前阶段暂时使用**无空格全拼**。例如 `nihao`、`zhongguo`。
- **word**: 对应的候选词或短语。
- **freq**: 基础词频。必须是整数，数值越大，该候选词在同拼音下的默认排名越靠前。

### 坏行处理规则

解析词库文件时，如果遇到不符合规范的行（坏行），将直接跳过该行，不抛出异常中断加载。符合以下任一情况即被视为坏行：

- **列数不对**: 拆分后的列数不等于 3。
- **空 pinyin**: `pinyin` 字段为空。
- **空 word**: `word` 字段为空。
- **freq 非整数**: `freq` 字段无法被解析为有效的整数。

### 当前阶段不支持的功能

当前版本的核心逻辑比较精简，因此暂不支持以下功能：

- **多音字**: 未做针对性处理。
- **拼音切分**: 不支持类似 `ni'hao` 的切分，仅支持连在一起的全拼。
- **模糊音**: 不支持类似 `z` <-> `zh` 的模糊匹配。
- **整句输入**: 目前仅针对词和短语。
- **词库压缩**: 词库直接以纯文本格式加载。

## 示例

以下是至少 10 条有效词库示例：

```text
nihao	你好	1000
zhongguo	中国	900
ceshi	测试	800
shurufa	输入法	700
shuju	数据	600
jihua	计划	500
yuyan	语言	400
shijian	时间	300
kongjian	空间	200
weilai	未来	100
banben	版本	50
```

## 未来扩展

未来可能会考虑对词库格式进行扩展，以支持更多的元信息。可能的扩展格式如下：

```
pinyin<TAB>word<TAB>freq<TAB>source<TAB>tags
```

其中新增字段说明：
- **source**: 词条来源（例如搜狗词库、用户自定义等）。
- **tags**: 词性或其他标签（例如 noun、verb）。
