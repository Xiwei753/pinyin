# Android Rime 无候选词排查指南

如果在测试中，你发现：
- 键盘底部显示了“中州韵（希）”或其他 Rime 方案的名称
- 输入拼音（如 `nihao`）时，预编辑区能正常显示英文字母
- **但是没有出现任何中文候选词**（如“你好”），甚至连候选窗口都没有弹出

这通常说明：
- Rime 引擎已经成功启动，并且加载了你的方案 (`schema` 配置文件基本正确被识别到了)
- **注意：如果能看到“中州韵（希）”，只能说明 schema 被识别，不代表词典已经部署成功。真正成功的标志是 `data/rime/build/` 生成部署产物，且输入 `nihao` 出现中文候选。**
- 问题出在：**词典未正确编译/找不到依赖**，或者 **Fcitx5 Android 的候选窗口设置有误**

*注：如果输入 `nihao` 出现了候选，但出现的是繁体/异体候选（如“妳好”、“逆號”），说明部署和编译已经成功，现在是进入了简繁过滤阶段。遇到这种情况，请直接下载并导入最新的修复版本，测试第一候选是否变回“你好”即可，无需再看本无候选排查指南。*

请按照以下顺序进行排查：

## 1. 检查候选窗口是否被隐藏/关闭

在部分手机的键盘上，候选词窗口可能会被折叠：
- 尝试点击候选栏右侧的**小箭头**或**展开按钮**，看看是否有被折叠的候选词。
- 检查 Fcitx5 Android 的键盘设置，确保“显示候选词”等相关选项是开启的。

## 2. 切换到内置方案进行排查 (关键步骤)

在本项目生成的配置文件中，我们为你添加了一个内置兜底方案（**明月拼音·简化字 / luna_pinyin_simp**）。

- “希为拼音” (xiwei_pinyin) 是我们的目标自定义方案。
- “明月拼音·简化字” (luna_pinyin_simp) 只是用于调试兜底，不是最终目标，它的主要作用就是排查问题。

请在输入法键盘中，**将当前方案切换为“明月拼音·简化字”**：
- 输入 `nihao`，观察是否有中文候选。

**结果分析：**
- **如果内置方案 `luna_pinyin_simp` 有候选词：**
  - 说明你的 Fcitx5 Android 的 Rime 引擎、候选窗口显示等一切正常。
  - 问题完全出在“希为拼音” (`xiwei_pinyin`) 自己的配置上。比如：词库中 `import_tables` 指定的依赖没有找到，或者词库文件被损坏。你可以检查 `xiwei_pinyin.dict.yaml` 是否正确配置。
- **如果内置方案 `luna_pinyin_simp` 也没有候选词：**
  - 说明问题出在 **Rime 部署环境、缓存或者 Fcitx5 的设置**，而不是你自己的方案文件写错了。
  - 因为 `luna_pinyin_simp` 是 Fcitx5 Rime 插件自带的非常成熟的方案，如果它都无法出词，说明 Rime 引擎根本没有成功编译任何词典。

## 3. 清理缓存并重新部署

如果你遇到了上面的第二种情况（所有方案都不出词），很可能是 Rime 在编译字典时产生了损坏的缓存。

**操作步骤：**
1. 打开文件管理器，导航到你之前导入数据的地方，比如对于 Fcitx5，通常是在：
   `Android/data/org.fcitx.fcitx5.android/files/data/rime/`
2. **删除该目录下的 `build/` 文件夹。** （包含所有的 `.bin` 文件）
3. 或者直接在 Fcitx5 的设置中重新导入一遍我们最新生成的 `android-rime-fcitx5-userdata.zip` 数据包。
4. **务必彻底重启 Fcitx 实例 / 重新部署**：
   - 可以在 Fcitx5 设置中找到 RIME 插件选项并点击“重新部署”。
   - 也可以在系统设置的“键盘与输入法”中将 Fcitx5 关闭后再重新开启。

再次测试内置方案和“希为拼音”，通常候选就能正常出现了。
